# 用户画像与协作指南

## 1. 工作模式与习惯

- 用户对系统功能要求高，注重细节准确性
- 喜欢通过截图沟通问题，直观高效
- 习惯用表格整理数据（Excel截图）
- 对计算逻辑敏感，会主动验证结果
- 倾向于在现有内容基础上迭代修改，而非从零开始

## 2. 业务领域

- **核心业务**：客服外包 / 拼席项目管理
- **涉及项目**：妲润、博思、讨师、森昊、悦泰（兼职）、益生菌等
- **关键需求**：
  - 月度结算与薪资计算
  - 节假日薪资计算（标准计薪 / 合作计薪）
  - 发票管理与开票信息维护
  - 阶梯定价与超店铺附加费计算
- **业务模式**：
  - **专席**：9个项目（定制商品、消防、晨光信息、晨光办公用品、福斯特、FDA注册、检测、模具、咨询）
  - **拼席**：5个项目（妲润、博思、讨师、森昊、悦泰兼职）

## 3. 技术偏好

- 使用网页系统（HTML/JS）管理数据
- 需要导出 Excel 和 PDF 结算单
- 数据存储在浏览器 IndexedDB 中（使用 Dexie.js）
- 系统部署在 GitHub Pages
- 结算逻辑包含三种类型：日均接待量阶梯、月总接待量阶梯、按时薪

## 4. 沟通风格

- 直接指出问题，要求精准修复
- 会多次验证直到完全正确
- 重视数据完整性和一致性
- 对系统美观度有要求
- 善于提供详细的上下文和截图辅助说明

## 5. 常见需求模式

| 需求类型 | 具体内容 |
|---------|---------|
| 项目配置修改 | 调整阶梯定价、节假日规则、税率、基础店铺数等 |
| 结算单生成 | 按月度生成各项目结算单并导出 Excel/PDF |
| 发票管理 | 维护开票信息、一键复制、区分含税/不含税 |
| 系统排查 | Bug 修复、计算逻辑校验、数据同步问题 |
| 数据更新 | 同步最新项目配置到 Excel 记忆文件 |

## 6. 注意事项

- **计算逻辑同步**：修改配置后必须确认计算逻辑同步更新，避免公式与配置脱节
- **数据同步保护**：数据同步不能覆盖用户自定义修改，需保留用户已填写的业务数据
- **截图识别精度**：截图识别可能有精度问题，关键数据需要用户确认
- **节假日计算**：节假日计算容易出错，涉及标准计薪与合作计薪两种模式，需特别仔细
  - 标准计薪：额外费用 = 日薪 × 计薪天数 × (倍数 - 1)
  - 合作计薪：额外费用 = 日薪 × 合作天数 × 倍数（直接乘，不额外减1）
  - 按假期名称分组，不重复计算
- **税务规则**：
  - 发票税点 = 0% → 含税，合计 = 小计
  - 发票税点 > 0% → 不含税，合计 = 小计 + 小计 × 税率
- **合同有效期**：修改合同起止日期时注意结算单的月份范围校验
