# 拼席月度结算管理系统 - 设计文档

## 1. 项目背景与目标

### 1.1 背景
公司为电商客服外包服务商，同时运营多个独立项目。各项目结算标准独立，主要按日均/月均接待量分档计费，且节假日计薪规则各不相同。目前使用Excel管理，计算繁琐且容易出错。

### 1.2 目标
搭建一个纯前端本地运行的拼席月度结算管理系统，实现：
- 各项目结算标准的配置与管理
- 月度接待量数据的录入与自动计算
- 节假日特殊结算规则的自动处理
- 历史数据留存与汇总分析展示
- 数据导入导出能力

## 2. 功能模块

### 2.1 项目管理模块
- 项目增删改查
- 每个项目独立配置：
  - 结算档位（接待量区间 -> 月服务费）
  - 店铺数量限制及超量附加费规则
  - 工作时间
  - 月休天数
  - 法定节假日计薪标准（额外倍数或天数）
  - 加班计薪标准
  - 发票税点
  - 工作内容说明

### 2.2 月度录入模块
- 选择月份 + 项目
- 日历形式展示当月，逐日录入接待量
- 实时显示日均接待量、月总接待量
- 自动匹配对应结算档位
- 支持从Excel批量导入整月数据

### 2.3 节假日处理
- 内置中国法定节假日数据（元旦、春节、清明、劳动节、端午、中秋、国庆）
- 根据各项目配置的节假日规则自动计算额外费用
- 支持手动调整当月实际法定节假日本公司安排

### 2.4 结算单生成
- 月度结算明细：
  - 基础服务费（按档位）
  - 节假日额外费用
  - 超店铺附加费
  - 发票税额
  - 合计金额（含税/不含税）
- 支持导出Excel/PDF

### 2.5 Dashboard汇总
- 各项目当月费用汇总卡片
- 月度费用趋势图（近12个月）
- 各项目接待量对比图
- 年度累计费用统计

### 2.6 历史查询
- 按项目 + 月份查询历史结算单
- 支持对比不同月份数据

## 4. 结算计算逻辑

### 4.1 基础服务费
1. 计算当月日均接待量 = 月总接待量 / 当月实际工作天数
2. 或直接使用月总接待量
3. 根据项目类型匹配对应档位，获取基础月费

### 4.2 节假日额外费用
- 若规则为"额外双倍薪资"：节假日当天费用 = 日均费用 * (倍数 - 1)
- 若规则为"额外天数"：节假日期间按合作天数额外计薪

### 4.3 超店铺附加费
- 若实际店铺数 > 限制：附加费 = (实际 - 限制) * 单店附加费

### 4.4 发票税额
- 含税金额 = 小计 * (1 + 发票税率)

## 5. 页面结构

| 页面 | 功能 |
|------|------|
| Dashboard | 月度总览、费用汇总卡片、趋势图表 |
| 项目管理 | 项目列表、结算标准配置、增删改 |
| 月度录入 | 日历录入、批量导入、实时计算 |
| 结算单 | 明细展示、导出功能 |
| 历史查询 | 月份筛选、对比查看 |

## 6. 技术选型

- **架构**：纯前端单页面应用（SPA），无后端依赖
- **存储**：IndexedDB（Dexie.js封装）
- **UI**：原生HTML/CSS/JS + 轻量CSS框架
- **图表**：Chart.js
- **Excel导入导出**：SheetJS (xlsx)
- **部署**：单个HTML文件，浏览器打开即用

## 7. 约束与假设

- 数据仅保存在本地浏览器，清除缓存会丢失，需定期导出备份
- 节假日数据内置2024-2028年，之后可手动添加
- 各项目结算档位线性递增，系统按区间匹配
